# QUESTÕES PARA O QUIZ

## O que o comando git add faz?
````markdown 
- Cria um novo branch

- Envia as alterações para o GitHub

- Adiciona arquivos à área de staging

- Salva permanentemente as alterações no histórico
````
## Qual cenário representa corretamente o uso de git pull e git push?
````markdown 
- Pull envia alterações locais, push recebe alterações

- Pull sincroniza alterações do remoto para o local, push envia alterações locais para o remoto

- Ambos fazem a mesma operação

- Push só funciona após pull obrigatório sempre.
````

## Seu git push foi rejeitado. O que pode ter acontecido?
````markdown 
- O Git bloqueia pushes automaticamente após múltiplos commits consecutivos

- Repositório local está desatualizado

- O arquivo enviado ultrapassa um limite interno do Git local

- O branch foi excluído automaticamente após o último commit
````

## O que o git status permite identificar?
````markdown 
- Apenas commits anteriores

- O estado atual dos arquivos

- Alterações no repositório remoto

- Conflitos resolvidos automaticamente
````

## Você acabou de alterar um arquivo. Quais comandos você usa até subir essa alteração?
````markdown 
- git commit -m "msg" → git add . → git push

- git add . → git commit -m "msg" → git push

- git push → git add → git commit

- git init → git add → git push
````

## O que faz o comando git config --global user.name "Seu Nome"?
````markdown 
- Define o nome do usuário apenas para o repositório atual

- Define o nome do usuário para todos os repositórios do sistema

- Cria um novo usuário no GitHub

- Altera o nome do repositório remoto
````

## Um arquivo aparece como “modified” no git status. O que isso significa?
````markdown 
- Ele foi alterado, e já foi adicionado à staging área

- Ele já está incluído no próximo commit

- Ele foi enviado para o repositório remoto

- Ele foi alterado, mas ainda não foi adicionado à staged área
````

## Durante o fluxo de versionamento no Git, qual comando move alterações do estado “modified” para “staged”?
````markdown 
- git commit

- git add

- git merge

- git push
````

## Ao executar git remote add origin <url>, o que muda no repositório local?
````markdown 
- O Git passa a enviar automaticamente todos os commits para o repositório remoto sem necessidade de outros comandos

- O histórico do repositório remoto substitui o histórico local automaticamente

- O repositório local é desvinculado de qualquer repositório remoto previamente configurado

- Uma referência a um repositório remoto é criada com o nome “origin”, sem enviar ou baixar dados ainda
````

## O que o comando git diff permite analisar antes de realizar um commit?
````markdown 
- O histórico completo de commits do projeto, incluindo autor, data e mensagens associadas 

- As diferenças exatas entre o estado atual dos arquivos e o último commit ou staging area 

- Quais arquivos já foram enviados para o repositório remoto 

- A estrutura completa de branches existentes no projeto
````

## Ao utilizar git switch, o que você está fazendo no fluxo de trabalho com Git?
````markdown 
- Alterando o histórico de commits do projeto para refletir mudanças mais recentes 

- Mudando o contexto de trabalho para outra branch ou iniciando uma nova branch a partir da atual 

- Enviando automaticamente alterações para o repositório remoto sem necessidade de outros comandos 

- Comparando diferenças entre versões de arquivos em diferentes commits
````

## Qual comando permite visualizar tanto as branches locais quanto as remotas?
````markdown 
- git branch 

- git branch -a 

- git remote 

- git fetch
````
